Features

Everything you need

A complete calibration toolkit with real-time visual feedback at every step.

◎

Live Detection

ChArUco corners and ArUco markers detected and drawn on every frame in real time. Supports OpenCV 4.8+ API with automatic legacy fallback.

★

Frame Scoring

Each frame scored on corner count, hull spread, new coverage, and sharpness. Rejected frames show the reason in red — no bad data gets through.

▣

Coverage Tracking

Grid coverage, quadrant coverage, scale diversity, and view diversity. A unified quality meter tells you when you have enough data.

◑

Corner Heatmap

Gaussian-splat heatmap shows observation density across the image. Instantly see dead zones where you need more board positions.

⏸

Auto-Capture

Toggle auto-capture mode and just move the board. Frames are accepted automatically when they meet quality thresholds with configurable cooldown.

⚙

ROS 2 Ready

Use a simple --ros-topic flag or the dedicated ROS 2 package with full parameter support. All settings exposed as ROS parameters.

Scoring

Smart frame selection

Every frame is evaluated on four weighted criteria before it's accepted into the calibration dataset.

Component Weight What it measures
Corner ratio 0.30 Fraction of possible ChArUco corners detected
Hull spread 0.20 Convex hull area of detected corners relative to image area
New coverage 0.30 How many previously-empty grid cells this frame would fill
Blur norm 0.20 Image sharpness via Laplacian variance — higher is better

Configuration

Fully configurable

All parameters via YAML config file with CLI overrides. Measure your board, set the values, and go.

default_config.yaml
board:
  squares_x: 9           # squares across (horizontal)
  squares_y: 7           # squares down (vertical)
  square_length: 0.04    # metres
  marker_length: 0.03    # metres (must be < square_length)
  aruco_dict: DICT_4X4_50

thresholds:
  min_corners: 6
  min_blur_var: 50.0
  min_score: 0.3
  capture_cooldown_ms: 1500

source:
  camera_id: 0
  width: 0              # 0 = native resolution
  height: 0
Integration

ROS 2 support

Two integration paths depending on your workflow.

Quick

CLI with --ros-topic

Pass a ROS 2 image topic to the standard CLI. Lazily imports rclpy — zero ROS dependency at install time.

charuco-calibrate \
  --ros-topic /camera/image_raw
Recommended

Dedicated ROS 2 Package

Full ROS 2 node with all settings as ROS parameters. Build with colcon, launch with ros2 launch.

colcon build --packages-select \
  charuco_calibrator_ros
source install/setup.bash

ros2 launch charuco_calibrator_ros \
  calibrator.launch.py \
  image_topic:=/camera/image_raw
